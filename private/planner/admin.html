Setup & Troubleshooting

Initial Setup

  1. Azure App Registration: Create an app with delegated permissions for Tasks.ReadWrite, Group.ReadWrite.All, and User.Read.
  2. Update config.json: Add clientId, authority, and allowedTenants.
  3. Set Admin Access: Either assign the PlannerAdmin app role or add emails to Admin Users list.
  4. Create/Link Plan: Enter your Microsoft Planner plan ID in Configuration section.

Known Issues

HTTP 429 (Too Many Requests)

Microsoft Graph enforces throttling limits. The app automatically retries with exponential backoff (up to 5 attempts). If you see 429 errors:

  • Wait a few minutes before retrying
  • Avoid rapid bulk operations (theme syncs, admin list updates)
  • Monitor the status badge for retry progress

Admin Authorization Failures

If you cannot access the admin portal:

  • Verify your email is in the Admin Users list (check config.json)
  • If using Entra group, confirm membership in the specified Admin Group ID
  • Check that your account has the PlannerAdmin app role assigned
  • Try signing out and signing back in to refresh token claims

Theme Sync Issues

If theme names don't sync to Planner:

  • Ensure the Plan ID is correct and you have write access
  • Check that all 7 theme names are defined in the theme section
  • Click "Save & Sync" to push changes to Planner
  • Wait for the green success status before closing the modal

Common Tasks

  • Change Plan: Update Plan ID and click "Save & Sync"
  • Update Admin List: Modify Admin Users emails, then "Save & Sync"
  • Sync Themes: Edit theme names and click "Save & Sync" to update Planner immediately
  • Reset Local Settings: Click "Reload Local Settings" to discard unsaved changes
  • View Config JSON: Click "Save & Sync" to see the generated config format
